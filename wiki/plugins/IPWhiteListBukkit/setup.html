1. Important configuration:

The configuration file is really small and most of it are messages for commands. Please see configuration for further reference. Anyways, the most important part of the configuration which needs to be set up is:

#Connecting to bungeeCord:
connect:
  #Type of the connection (YML - for connecting through files as it's been till now, SOCKET - using sockets):
  type: SOCKET
  #Full path to data.yml file in IPWhiteList's folder (on BungeeCord):
  #NOTE! You can use ../ strings at the beginning of the path to go one folder up (begins at folder where this plugin's .jar file is).
  #      As directory and file separator please use \\ (on Windows) or / (on Linux).
  bungeePath: 'home\BungeeCord\plugins\IPWhiteList\data.yml'
  #Socket configuration:
  socket:
    #IP of the server (IP of the machine where is server with IPWhiteList plugin hosted, or 127.0.0.1 if on the same machine):
    ip: 127.0.0.1
    #Port of the server (as set in IPWhiteList's config.yml):
    port: 4444

2. Connecting to the IPWhiteList:

The first few lines with the section name "connect" are configuration of connection with the IPWhiteList. If you don't have it installed, go and do it.

type: There you need to write the type of the connection. You have two options, YML and SOCKET .

  • YML - The file communication is old, hard to setup for some people and may be slow, but is really reliable. This way of communication is provided by extension obtaining data.yml file which is generated in the IPWhiteList's plugin folder.

  • SOCKET - Communication is provided over internet, it's fast and supports MultiCraft, multi-machine servers and is comfortable to be set up. The only problem is it can cause delays, mainly if using between two different messages, for example when internet connection failes, however, this should not happen.

3. Path to the BungeeCord:

Please read about this type of communication above. If you selected this communication, you must set the system path to the data.yml file which is located in the IPWhiteList's plugin folder at the "bungeePath" tag.

Please mind following:

  • Path needs to be full system path - This means path can be C:\abc\def\data.yml, home/abc/def/data.yml, etc...

  • Parent folder ../ code is supported - At the beggining of the path you can specify ../ code which will go to the upper (parent) folder. If used, the path begins in the /plugins/ server folder. For example, you have IPWhiteListBukkit located in home/Spigot/plugins/ and IPWhiteList data.yml file in home/BungeeCord/plugins/IPWhiteList/data.yml. You start in home/Spigot/plugins/ folder. You would use first ../, you would go from home/Spigot/plugins/ to home/Spigot/. After that you would use the second one, which would get you to the home/ folder. And now you are in folder where the BungeeCord is. So add that part BungeeCord/plugins/IPWhiteList/data.yml to the path. The path should look like ../../BungeeCord/plugins/IPWhiteList/data.yml. Here you go!

  • Different OS, different file separator - If using Windows, please use \\ file separator (example usage: C:\\Desktop\\abc.yml), if using Linux, Debian or Ubuntu, please use / (example usage: home/abc/def.yml). Also, \ file separator should be working, but it's not guaranteed to work everywhere.

4. Socket connection:

If you chose the socket communication, you need to configure two things under the section named "socket".

ip: The IP on which is is BungeeCord server hosted on, or 127.0.0.1 if hosted on the same machine.

port: The port on which is IPWhiteList's socket server hosted on, by default 4444. Must be the same as socketPort in config.yml of IPWhiteList; learn more here.
